# HD CyCIF NOX1 analysis pipeline

Image-analysis code for quantifying **NOX1** in **hepatocytes** from single-cycle 6-plex cyclic
immunofluorescence (CyCIF) liver imaging: illumination re-correction / de-seaming → autofluorescence
removal → cross-sample normalization → watershed segmentation → hepatocyte typing → NOX1
quantification and per-scene review figures.

See [`METHODS.md`](METHODS.md) for the manuscript methods text with all parameters.

## Pipeline

| Step | Script | Does | Key output (per sample, under `<HD_DATA_ROOT>`) |
|---|---|---|---|
| 1 | `pipeline/1_deseam_refuse.py` | Per-sample BaSiC flat-/dark-field on raw tiles, tile-position detection, overlap balancing, feathered re-fusion of all 6 channels; then NNLS-v3 autofluorescence removal | `<sample>/AF_removal/fused_decon_refused.ome.tif`, `NOX1_refused.tif` |
| 2 | `pipeline/2_crosssample_norm.py` | Additive NOX1 normalization across samples (ECM-background anchor) | `<sample>/AF_removal/NOX1_refused_norm.tif` |
| 3 | `pipeline/3_segment.py` | Watershed nuclei + tissue-filling cell bodies + membrane ring; per-cell 6-channel quantification | `<sample>/segmentation_v4/{whole_cell_label,nuclei,membrane_ring}_v4.tiff`, `cell_stats_v4.csv` |
| 4 | `pipeline/4_celltype_quantify.py` | Cell typing (hepatocyte = ASGR1⁺ & CD45⁻), NOX1 normalization, per-patient/sample **median** NOX1 | `_nox1_normalization/cell_stats_v4_gated.parquet`, `nox1_v4_per_{sample,patient}.csv` |
| 5 | `pipeline/5_export_excel.py` | Multi-sheet Excel of the hepatocyte NOX1 median (CD45⁺ reference) | `_nox1_normalization/nox1_v4_results.xlsx` |
| 6 | `pipeline/6_scene_figures.py` | High-resolution 2×2 review JPG per scene (DAPI+NOX1, NOX1, NOX1+hepatocyte-mask, ASGR1) | `_nox1_normalization/scene_figures/<sample>.jpg` |

Shared helpers live in the importable package `hdcycif/` (`config`, `tiles`, `af_removal`,
`segmentation`); the numbered `pipeline/` scripts import only from `hdcycif` and are run in order.

## Installation

```bash
conda env create -f environment.yml && conda activate hdcycif
# or:  pip install -r requirements.txt   (Python 3.10)
```

Exact pinned versions (BaSiCPy 1.2.0 / JAX 0.4.23, scikit-image, scikit-learn, SciPy, tifffile,
NumPy, pandas, Matplotlib, Pillow, openpyxl) are in `requirements.txt`. The pipeline is CPU-only.

## Running

Set `HD_DATA_ROOT` to the dataset root (the folder containing the per-scene `sample_*` directories)
and run the steps in order:

```bash
export HD_DATA_ROOT=/path/to/HD          # Windows PowerShell:  $env:HD_DATA_ROOT = "D:\CycIF\HD"
python pipeline/1_deseam_refuse.py
python pipeline/2_crosssample_norm.py
python pipeline/3_segment.py
python pipeline/4_celltype_quantify.py
python pipeline/5_export_excel.py
python pipeline/6_scene_figures.py
```

Optional environment variables: `HD_SAMPLES` (comma-separated subset of samples), `HD_FIG_DS`/
`HD_FIG_DPI`/`HD_FIG_IN`/`HD_FIG_JPGQ` (figure resolution/quality), `HD_MAKE_PDF=1` (also assemble
the scene JPGs into one PDF). Outputs are written under `<HD_DATA_ROOT>` as listed above.

### Inputs
Step 1 reads the raw exported single tiles (`<sample>/cyc001/Z-Stacks/fileseries_export/z01/tiles/`)
and uses the upstream DAPI mosaic (`<sample>/AF_removal/…nnls_v3.ome.tif`) only as the coordinate
frame for tile-position detection. Cross-sample normalization uses the ECM mask
(`<sample>/segmentation_v2/ecm_label.tiff`). These upstream products are generated by the acquisition
pipeline (Cycif_pipeline_V3, referenced), not by this repository.

## Data availability
This repository contains **code only**. Raw and processed imaging data are not included (size and
patient confidentiality) and are available from the authors on reasonable request / from the
indicated archive.

## Notes
- Segmentation uses a watershed approach (`pipeline/3`) rather than a deep-learning model because the
  analysis was run CPU-only, where CellposeSAM is impractically slow; the tissue-filling cell bodies
  are tuned so the ASGR1 membrane is captured for hepatocyte typing.
- Upstream acquisition and preprocessing (illumination, deconvolution, EDF, stitching, and the
  original AF removal) were performed with an in-house pipeline (Cycif_pipeline_V3) and are cited,
  not reproduced here.
